The Project

 

Overview

The project is a synchronisation program called 'Drag-on', that gets browser tabs, files, any strings of texts in the clipboard from device to device by dragging across, holding and swiping and using clipboard shortcuts. The idea here is to seamlessly move data from your computer to your phone or device as easily as it is to move a paper note around and appear as though there's a virtual paper inside the devices. Drag-on will eventually include features for non digital data such as OCR and hand writing recognition. The Data will be encrypted so it can't be intercepted.

 

Motivation

Why is this important? Currently we still print out documents and carry small bits of paper around with us. If we want a paperless world we need to find more convenient ways to move data around. Most of the time while trying to get photos and documents off my phone, I had to use cables or send smaller files via emails to myself, even small things like copy and pasted text and numbers are annoying to get from one digital device to another. If you talk to most people, they'll describe similar experience moving things from device to device.

 

Description

Drag-on is a synchronisation service that helps people get data from one device to another locally via Wi-Fi and Bluetooth. The idea to make it easier for technical and nontechnical people to get browsers tabs, clipboard large files computer their phone and move to eventually written data, enabling people to copy and paste from real world sources like someone's hand writing data as well as books and character based printed sources.

The user must create and account so that Drag-on knows who owns the devices and as a proof of purchase

The interface for Windows will appear when a shortcut key combination is pressed (which can also be changed by the user). It will show a pane with a QR code to link the device together (there will also be an alphanumeric code that can be typed in should the code fail),  For clipboard functionality there is also a additional options added to right click menus which include: Copy-to-phone and Paste-to-phone. The Windows interface will offer a means for grabbing files from the phone without having to do anything else.

 

The interface for Android phone will open as an application which once setup will run in the background until closed at the users request. The application will use the camera to read the QR code and determine what its looking for. If one of the two devices aren't owned or registered with the same account, it will ask you to use a second QR or password for use as a key. You can then save that user as a friend or trusted link. The interface, will give allow you to select files to transfer to another device once the link is established.  The android interface will offer browser addons for sending tabs back to other devices such as a Windows PC or another phone, but will also share the clipboard.

In order for a seamless user experience, Drag-on will have 2 modes which can be switched. A Technical mode for people with computer experience and a Non-Technical (AKA 'Just Work please') for people who need help. Both modes can be swapped if necessary. Drag-on will keep logs of all transfers so people can see where their data has gone and all data transfers are encrypted so that information isn't intercepted.

 

While Drag-on will start as a more intuitive way of synchronising PCs with mobile devices, but it will evolve into a way of dealing with everyday data.

There a number of future features that would be will later be implemented such as the ability to read text (OCR and handwriting recognition) instantly and reliably so all people have to do is point their phone at something and add that data to their device. There will also be a search function and eventually there will be a mode called, 'Give me', this for any data such as recepts or other small documents which the user would receive from everyday interaction with businesses and organisations and send them via NFC and Bluetooth.

 

Tool and Technologies

This would work with a combination of different software technologies. like to use normal Wifi and Adhoc(such as whats used to connect cameras phone tegther), Bluetooth and QR codes. AES 256 Encryption would be the ideal choice, but lesser options will be taken into consideration if need be. A QR code generator will be required and eventually handwriting recognition and OCR will be require in later versions. A user database will be required for security and purchasing methods
